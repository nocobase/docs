# 1Panel 面板安装的升级

## 1. 进入可升级页面

1. 登录 1Panel 管理面板
2. 点击左侧菜单 **「应用商店」**
3. 选择 **「可升级」** 标签页
4. 在应用列表中找到 NocoBase 应用

## 2. 执行升级操作

点击 NocoBase 应用右侧的 **「升级」** 按钮，在升级对话框中进行以下配置：

- **目标版本**：选择要升级到的 NocoBase 版本
- **升级前备份应用**：
  - ✅ 推荐启用此选项
  - 升级失败时系统会自动使用备份回滚
  - 回滚日志可在 **「日志审计」→「系统日志」** 中查看
- **自动拉取镜像**：
  - ✅ 默认启用
  - 在应用启动前自动执行 `docker pull` 拉取最新镜像
  - 确保获取到最新的应用文件

## 3. 文件变更确认

点击 **「查看对比」** 按钮，检查配置变更：

- 升级操作会替换 `docker-compose.yml` 文件
- 对比新旧配置文件的差异
- 如有手动修改的配置，请决定是否保留更改

## 4. 开始升级

确认所有配置无误后，点击 **「确认」** 按钮，系统将开始自动升级流程：

- 执行应用备份（如启用）
- 拉取新的 Docker 镜像
- 停止旧版本容器
- 启动新版本容器
- 验证服务状态

升级后验证，确认服务状态：

- 进入 **「已安装」** 页面，确认 NocoBase 容器状态为「运行中」
- 查看应用日志，确认无错误信息
- 访问 NocoBase 应用，验证功能正常

## 5. 故障处理

如在升级过程中出现异常，系统会 **自动回滚**  到升级前的状态：

- 若启用了 **备份功能**，系统将在升级失败后自动恢复至升级前状态
- 可前往 **「日志审计」** → **「系统日志」** 查看具体的失败原因

你也可以 **手动恢复**：

- 进入 **「备份」** 页面
- 选择最近的备份文件执行恢复操作
